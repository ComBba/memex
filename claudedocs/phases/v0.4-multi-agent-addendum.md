## 2. Phase 1 (Security) · v0.4 amendments

### 2.1 SandboxRoot becomes multi-root

```rust
// src-tauri/src/indexer/sec.rs (P1 module)
pub struct SandboxRoot {
    canonical_roots: Vec<PathBuf>,   // [~/.claude/projects, ~/.codex/sessions]
}

impl SandboxRoot {
    pub fn from_env() -> anyhow::Result<Self> {
        let home = std::env::var("HOME")?;
        let claude_root = PathBuf::from(&home).join(".claude/projects");
        let codex_root  = PathBuf::from(&home).join(".codex/sessions");
        let canonical_roots: Vec<PathBuf> = [claude_root, codex_root]
            .into_iter()
            .filter_map(|p| p.canonicalize().ok())     // skip missing roots (graceful)
            .collect();
        if canonical_roots.is_empty() {
            anyhow::bail!("no valid session root found (neither ~/.claude/projects nor ~/.codex/sessions exists)");
        }
        Ok(Self { canonical_roots })
    }

    pub fn contains(&self, p: &Path) -> anyhow::Result<PathBuf> {
        let canon = p.canonicalize()?;
        for root in &self.canonical_roots {
            if canon.starts_with(root) {
                return Ok(canon);
            }
        }
        anyhow::bail!("path outside sandbox: {}", canon.display());
    }

    pub fn detect_agent(&self, p: &Path) -> Option<SourceAgent> {
        // first matching root wins
        let canon = p.canonicalize().ok()?;
        let home = std::env::var("HOME").ok()?;
        let claude_prefix = PathBuf::from(&home).join(".claude/projects").canonicalize().ok()?;
        let codex_prefix  = PathBuf::from(&home).join(".codex/sessions").canonicalize().ok()?;
        if canon.starts_with(&claude_prefix) { Some(SourceAgent::ClaudeCode) }
        else if canon.starts_with(&codex_prefix) { Some(SourceAgent::Codex) }
        else { None }
    }
}

pub enum SourceAgent { ClaudeCode, Codex }
```

## 4. Codex Parser (`codex_parser.rs`) · NEW module

### 4.1 Location

`src-tauri/src/codex_parser.rs` — peer to `parser.rs`. Both expose the same `parse_session(path) -> Result<Session>` signature so indexer.rs is parser-agnostic.

### 4.2 Schema reference

See memory file `codex_cli_session_schema.md` for line-type reference. Summary:

- Every line: `{timestamp: ISO 8601, type: "session_meta"|"event_msg"|"response_item", payload: {...}}`
- `session_meta` is exactly once at file head — carries `id`, `cwd`, `originator`, `cli_version`, `source`, `model_provider`
- `response_item` with `payload.role` = `user | assistant | developer` = turns
- `response_item` with `payload.type` = `function_call` = tool calls
- `response_item` with `payload.type` = `function_call_output` = tool outputs (check for error markers)

### 4.3 Mapping to common `Session` struct

| Common field | Codex source |
|--------------|--------------|
| `session_id` | `session_meta.payload.id` |
| `project_path` | `session_meta.payload.cwd` (already absolute, no decoding) |
| `project_name` | basename(project_path) |
| `start_iso` | `session_meta.timestamp` |
| `end_iso` | last line's `timestamp` |
| `claude_version` | `null` (Codex doesn't set this) — use `cli_version` for `codex_version` parallel field, or omit |
| `user_turns` | count of `response_item` with `payload.role == "user"` |
| `assistant_turns` | count of `response_item` with `payload.role == "assistant"` |
| `tool_count` | count of `response_item` with `payload.type == "function_call"` |
| `has_errors` | any `function_call_output` whose `content` contains error markers (heuristic) |
| `git_branch` | `null` for Codex MVP (not in schema) |
| `ai_title` | derived from first user message snippet (heuristic) |

### 4.4 Test fixtures

`src-tauri/tests/fixtures/codex/`:
- `01_minimal_codex.jsonl` — session_meta + one user message + one assistant message
- `02_with_tools.jsonl` — adds 2 function_call + 2 function_call_output
- `03_with_errors.jsonl` — function_call_output containing error markers
- `README.md` — fixture rationale + how each was sanitized from real session
