§2 Tech Kicks · Qdrant SOTA의 끝

Qdrant 1.18.0 (2026-05-11 출시)의 최신 기능을 Memex의 invariant 안에서 끝까지 끌어 쓴다.

Single-call Multi-stage Query API + RRF

현재 lens_search는 5개의 parallel cosine query (indexer.rs:539-615). 이걸 Qdrant 1.10+ Query API의 prefetch + fusion: rrf로 옮긴다.

{
  "prefetch": [
    { "using": "content",      "query": <dense>, "limit": 50 },
    { "using": "path_sparse",  "query": <BM25>, "limit": 50 },
    { "using": "tool",         "query": <dense>, "limit": 50 },
    { "using": "error",        "query": <dense>, "limit": 50 }
  ],
  "query": { "fusion": "rrf" }, "limit": 20
}
Round-trips: 5 → 1 Effort: ~6h Sources: research/01 · research/04 §6

6번째 named vector content_late — late-interaction MaxSim

같은 BGE-small token-level 출력을 multivector point로. 새 모델 0.

"Late interaction models like ColBERT preserve token-level embeddings and compute MaxSim at query time." research/02 · qdrant.tech/articles/late-interaction-models/
nDCG@10 target: +15% [eval gate] Storage: +115 MB at N=10k

Server-side BM25 sparse on path & tool

Qdrant 1.15.2+ server-side tokenize. fastembed-rs 의존 없음.

MRR target: +30% Effort: ~3h

MMR diversification + Relevance Feedback

VSD brief "recommendation" 직격. Lens 결과에 diversity=0.4, Mix & Match에 👍/👎.

Scalar quantization + signed snapshots

6벡터 SQ → 4× 압축. Snapshot envelope에 SHA-256 + issuer.

Async ONNX (spawn_blocking) + cross-session embed batch=32

Embedder Mutex 해소. Cold scan 3-4× 가속.

Sources: analysis/05 QW-1 · research/05 §11
